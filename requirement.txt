我正在开发一个名为「五分钟健康刺客」的 Streamlit 应用，请按照以下所有要求进行开发。

一、用户系统要求

1. 应用需要支持多用户，每个用户有独立的用户名。
2. 用户在侧边栏输入用户名即可“登录/切换用户”（不需要密码，简化版）。
3. 所有健康记录按用户名分别存储。
4. 切换用户后，主页面显示的数据、历史记录都自动切换到对应用户。

二、UI 布局要求

侧边栏包含：
1. API Key 输入框（密码类型），用于调用 DeepSeek API
2. 城市输入框（用于获取天气）
3. 用户名输入框（登录/切换用户，不需要密码）
4. 历史记录展示区：显示当前用户最近 7 天的综合健康指数趋势图（折线图）
5. 可选：是否启用天气功能的开关

主页面包含：
1. 当前登录用户名称的显示
2. 五项健康指标输入（详见第三部分）
3. “召唤反击”按钮
4. AI 返回结果的展示区
5. 当前用户的历史记录列表（最近 5 条，显示日期和综合健康指数）

三、用户输入内容要求（主页面）

用户需要输入以下五项生活指标：

1. 久坐最长时段（选项：<1小时 / 1-3小时 / 3-5小时 / >5小时）
2. 低头累计时长（选项同上）
3. 睡眠时长（滑动条，3-10小时，步长0.5）
4. 今日饮水量（选项：<500ml / 500-1000ml / 1000-1500ml / >1500ml）
5. 今日压力自评（滑动条，1-10分，1为无压力，10为极大压力）
四、外部数据要求

1. 根据用户输入的城市名（侧边栏），调用免费天气 API 获取当地实时天气。
2. 推荐的免费天气 API：wttr.in（无需 Key）或 WeatherAPI（免费注册）。
3. 天气信息至少包含：天气状况（晴/雨/阴等）、气温（℃）、湿度（%）。
4. 如果天气 API 调用失败，要有提示并继续运行（使用默认天气“未知”）。

五、AI 回答内容要求

调用 DeepSeek API，回答必须包含以下四个部分，并用分隔线隔开：

第一部分：反击动作
- 给出一个 2-5 分钟可完成的动作
- 必须结合天气调整（例如：雨天推荐室内拉伸，晴天推荐去阳台晒背）
- 必须结合用户的久坐/低头/睡眠数据
- 动作要具体到步骤
第二部分：生活健康评分表
格式如下：
久坐健康分：xx/100（说明）
颈椎健康分：xx/100（说明）
睡眠质量分：xx/100（说明）
水分充足分：xx/100（说明）
压力管理分：xx/100（说明）
综合健康指数：xx/100


第三部分：刺客有话说
 一段像朋友聊天一样的自然语言评语（2-3 句话）
 风格：温暖 + 轻微幽默，不毒舌，不讽刺
 必须结合天气、睡眠、压力三个指标

第四部分：明日小贴士
 针对今天最薄弱的一项指标，给出一个明天可以做的具体改进建议

六、数据存储要求

1. 使用 SQLite 数据库，表名为 `health_records`。
2. 表结构如下：
   - id（主键，自增）
   - username（文本，用户名）
   - date（文本，记录日期，格式 YYYY-MM-DD）
   - sit（文本，久坐数据）
   - head（文本，低头数据）
   - sleep（浮点数，睡眠小时数）
   - water（文本，饮水量）
   - stress（整数，压力评分 1-10）
   - weather（文本，天气状况）
   - temperature（文本，气温）
   - ai_response（文本，AI 完整返回内容）
   - comprehensive_score（整数，综合健康指数 0-100）

3. 同一用户同一天只能有一条记录，重复提交时覆盖更新。
4. 数据库文件名为 `health_assassin.db`。

 七、历史记录展示要求

侧边栏：
显示当前用户最近 7 天的综合健康指数趋势图（使用 Plotly 或 Matplotlib 绘制折线图）
如果数据不足 7 天，按实际数据展示

主页面：
在 AI 结果展示区下方，显示当前用户的“最近 5 条记录”
每条记录显示：日期、综合健康指数、简短预览（如“久坐3-5小时，睡眠7h”）
点击某条记录，可以在下方展开查看当时的完整 AI 回复内容

八、API 调用要求

1. 使用纯 `requests` 库调用 DeepSeek API，不依赖 `openai` 库。
2. API 地址：`https://api.deepseek.com/v1/chat/completions`
3. 模型：`deepseek-chat`
4. 超时设置：10 秒
5. 失败时显示友好错误信息，不崩溃。

九、代码质量要求

1. 所有函数要有注释说明功能
2. 数据库操作要有异常处理
3. 页面布局整洁，使用 st.columns 等组件合理划分区域
4. 代码结构清晰，便于后续修改和扩展

十、可选加分项（如时间充裕）

1. 支持导出当前用户的所有历史记录为 CSV 文件
2. 添加“删除某条记录”功能
3. 支持用户修改当天的记录（覆盖更新）
4. 添加简单的数据统计（如最近 7 天平均综合健康指数）

 十一、输出要求

请输出完整的 `app.py` 代码，可以直接复制到 Streamlit 环境中运行。代码中需要包含所有必要的 import 语句和注释。
